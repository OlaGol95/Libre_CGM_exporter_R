# SQL Data Dictionary for LIBRE CGM and `iglu`

This document defines the database-facing meaning of the LIBRE/CGM export
generated by `program_LIBRE_CGM.py`.

The CGM workbook contains raw Libre glucose measurements split by visit and a
per-visit summary enriched with `iglu` metrics when R and the `iglu` package are
available.

## Scope

This dictionary covers only LIBRE/CGM data:

- glucose measurements,
- visit windows,
- Python glucose summary metrics,
- R `iglu` metrics.

## Recommended SQL Model

```sql
CREATE TABLE cgm_measurements (
    patient_id      TEXT NOT NULL,
    visit           TEXT NOT NULL,
    timestamp       DATETIME NOT NULL,
    glucose         REAL NOT NULL,
    source_file     TEXT,
    PRIMARY KEY (patient_id, visit, timestamp, glucose)
);

CREATE TABLE cgm_visit_summary (
    patient_id      TEXT NOT NULL,
    visit           TEXT NOT NULL,
    n               INTEGER,
    start           DATETIME,
    end             DATETIME,
    mean_glucose    REAL,
    median_glucose  REAL,
    sd_glucose      REAL,
    min_glucose     REAL,
    max_glucose     REAL,
    pct_below_54    REAL,
    pct_below_70    REAL,
    pct_70_180      REAL,
    pct_above_180   REAL,
    pct_above_250   REAL,
    PRIMARY KEY (patient_id, visit)
);
```

All `iglu` columns listed below can be added to `cgm_visit_summary` as nullable
columns. Use `REAL` for numeric metrics and `TEXT` or `DATETIME` for the two
active-period timestamp columns.

Valid visit values:

```text
v1, v3, v4, v6
```

Glucose unit:

```text
mg/dL
```

Percentage columns are stored on the `0-100` scale, not as fractions.

## Workbook Structure

Expected workbook:

```text
<patient_export_code>_CGM.xlsx
```

Worksheets:

```text
v1
v3
v4
v6
summary
```

## Visit Worksheets

Each visit worksheet has the same structure.

| Column | SQL type | Meaning |
|---|---:|---|
| `patient_id` | `TEXT` | Normalized patient ID. |
| `visit` | `TEXT` | Lower-case visit label: `v1`, `v3`, `v4`, `v6`. |
| `timestamp` | `DATETIME` | LIBRE glucose measurement timestamp. |
| `glucose` | `REAL` | Glucose value in `mg/dL`. |
| `source_file` | `TEXT` | Source/export identifier. |

Rows are selected from raw LIBRE CSV files by the visit windows stored in
`data_visits.xlsx`.

## Summary Worksheet: Python Metrics

These columns are produced by Python from the selected glucose values.

| Column | SQL type | Meaning |
|---|---:|---|
| `patient_id` | `TEXT` | Normalized patient ID. |
| `visit` | `TEXT` | Visit label. |
| `n` | `INTEGER` | Number of glucose measurements in the visit window. |
| `start` | `DATETIME` | First glucose timestamp in the visit. |
| `end` | `DATETIME` | Last glucose timestamp in the visit. |
| `mean_glucose` | `REAL` | Arithmetic mean glucose, `mg/dL`. |
| `median_glucose` | `REAL` | Median glucose, `mg/dL`. |
| `sd_glucose` | `REAL` | Sample standard deviation of glucose, `mg/dL`. |
| `min_glucose` | `REAL` | Minimum glucose, `mg/dL`. |
| `max_glucose` | `REAL` | Maximum glucose, `mg/dL`. |
| `pct_below_54` | `REAL` | Percentage of readings below `54 mg/dL`. |
| `pct_below_70` | `REAL` | Percentage of readings below `70 mg/dL`. |
| `pct_70_180` | `REAL` | Percentage of readings in `70-180 mg/dL`, inclusive. |
| `pct_above_180` | `REAL` | Percentage of readings above `180 mg/dL`. |
| `pct_above_250` | `REAL` | Percentage of readings above `250 mg/dL`. |

## Summary Worksheet: `iglu` Metrics

The following fields are produced by `run_iglu_summary()` using the R package
`iglu`. Store them as nullable `REAL` unless noted otherwise.

| `Active_percent_start_date` | `TEXT` or `DATETIME` | Start timestamp used by `iglu::active_percent()`. |
| `Active_percent_end_date` | `TEXT` or `DATETIME` | End timestamp used by `iglu::active_percent()`. |
| `ADRR` | `REAL` | Average Daily Risk Range. |

| `CONGA` | `REAL` | Continuous Overall Net Glycemic Action. |
| `CV` | `REAL` | Coefficient of variation for glucose. |
| `CV_subtypes_CVmean` | `REAL` | Mean component from `iglu::cv_measures()`. |
| `CV_subtypes_CVsd` | `REAL` | SD component from `iglu::cv_measures()`. |
| `eA1c` | `REAL` | Estimated HbA1c. |
| `GMI` | `REAL` | Glucose Management Indicator. |
| `GRADE` | `REAL` | Glycemic Risk Assessment Diabetes Equation score. |
| `GRADEeu` | `REAL` | Euglycemia component of GRADE. |
| `GRADEhyper` | `REAL` | Hyperglycemia component of GRADE. |
| `GRADEhypo` | `REAL` | Hypoglycemia component of GRADE. |
| `GVP` | `REAL` | Glycemic Variability Percentage. |
| `HBGI` | `REAL` | High Blood Glucose Index. |
| `LBGI` | `REAL` | Low Blood Glucose Index. |
| `Hyper_Index` | `REAL` | Hyperglycemia index. |
| `Hypo_Index` | `REAL` | Hypoglycemia index. |
| `IGC` | `REAL` | Index of Glycemic Control. |
| `IQR` | `REAL` | Interquartile range of glucose. |
| `J_index` | `REAL` | J-index glycemic variability metric. |
| `MAD` | `REAL` | Mean absolute difference. |
| `MAG` | `REAL` | Mean absolute glucose change. |
| `MAGE` | `REAL` | Mean amplitude of glycemic excursions. |
| `Mean` | `REAL` | Mean glucose from `iglu::mean_glu()`. |
| `Median` | `REAL` | Median glucose from `iglu::median_glu()`. |
| `MODD` | `REAL` | Mean of Daily Differences. |
| `M_value` | `REAL` | M-value glycemic control metric. |
| `Percent_Above_above_140` | `REAL` | Percentage of readings above `140 mg/dL`. |
| `Percent_Above_above_180` | `REAL` | Percentage of readings above `180 mg/dL`. |
| `Percent_Above_above_250` | `REAL` | Percentage of readings above `250 mg/dL`. |
| `Percent_Below_below_54` | `REAL` | Percentage of readings below `54 mg/dL`. |
| `Percent_Below_below_70` | `REAL` | Percentage of readings below `70 mg/dL`. |
| `Percent_In_Range_in_range_63_140` | `REAL` | Percentage of readings in `63-140 mg/dL`. |
| `Percent_In_Range_in_range_70_180` | `REAL` | Percentage of readings in `70-180 mg/dL`. |
| `Quantiles_0` | `REAL` | Minimum glucose quantile. |
| `Quantiles_25` | `REAL` | 25th percentile glucose. |
| `Quantiles_50` | `REAL` | 50th percentile glucose. |
| `Quantiles_75` | `REAL` | 75th percentile glucose. |
| `Quantiles_100` | `REAL` | Maximum glucose quantile. |
| `Range` | `REAL` | Glucose range: maximum minus minimum. |
| `ROC_mean_abs` | `REAL` | Mean absolute rate of glucose change. |
| `SD_of_ROC` | `REAL` | Standard deviation of rate of glucose change. |
| `SD` | `REAL` | Standard deviation from `iglu::sd_glu()`. |
| `SD_subtypes_SDw` | `REAL` | Within-day SD subtype from `iglu::sd_measures()`. |
| `SD_subtypes_SDhhmm` | `REAL` | Time-of-day SD subtype from `iglu::sd_measures()`. |
| `SD_subtypes_SDwsh` | `REAL` | Within-series/hour subtype from `iglu::sd_measures()`. |
| `SD_subtypes_SDdm` | `REAL` | Day-to-day mean subtype from `iglu::sd_measures()`. |
| `SD_subtypes_SDb` | `REAL` | Between-day SD subtype from `iglu::sd_measures()`. |
| `SD_subtypes_SDbdm` | `REAL` | Between-day mean subtype from `iglu::sd_measures()`. |

If a metric cannot be calculated, keep the SQL value `NULL`. If the exporter
creates an `iglu_status` column, store it as `TEXT`.

## Application SQLite Tables Relevant to LIBRE

`libre_uploads.sqlite3` is application state, not the analytical CGM database.
For LIBRE/CGM, the relevant internal tables are:

| Table | Purpose |
|---|---|
| `patient_status` | Tracks whether a patient is marked ready/exported in the GUI. |
| `patient_visit_flags` | Stores visit-level `ND` flags. |
| `libre_visit_metrics` | Stores visit-level LIBRE coverage and average glucose for the GUI. |
| `database_exports` | Tracks exported patients and target folders. |
